# Arush's Study Tracker

> A full-stack personal accountability system that converts YouTube comment likes into study hour obligations, syncs completed Todoist tasks to a GitHub-hosted session log via a Cloudflare Worker, and surfaces everything — live focus state, IST clock, 90-day heatmap, an hours-remaining trend graph, and real-time countdowns — in a single zero-dependency static page. A hidden, keybind-triggered admin panel lets the owner toggle focus/break and log completed sessions directly from the site.

## The Problem

Most study trackers are self-reported. You log what you want, skip what you don't, and the only person keeping score is you.

This one isn't.

**Hours owed** are generated by YouTube comment likes — a public, verifiable number that no one controls but the people watching. I leave comments on long study-with-me videos, and each like on those comments adds a fixed number of hours to my study obligation (e.g. a like on a 4-hour video comment adds 4 hours). **Hours completed** come from two sources: an automatic 5-minute sync of my completed Todoist tasks, and instant manual logging through a hidden admin panel on the site itself. The gap between owed and completed is displayed as a large number at the top of the page.

The result is a live accountability contract between me and anyone who visits. If you liked a comment, you contributed to the debt. If the hours aren't climbing, the deficit is public.

## How It All Fits Together

There are three moving parts, all version-controlled in this repo:

1. **`index.html`** — a static frontend with no framework and no build step. It fetches data from two sources, renders everything client-side, and hosts a hidden admin panel for the owner.
2. **`data.json`** (hosted on GitHub) — the completed side of the tracker. Stores total hours done and the full session log. Written to automatically by the Worker (on Todoist sync or manual log) via the GitHub Contents API.
3. **`worker.js`** (deployed to Cloudflare Workers) — a serverless edge function that sits between the browser and all external APIs. It holds every secret, handles caching, and exposes a set of `?action=` endpoints plus a couple of path-based shortcuts.

```
Browser  (index.html — no dependencies, no build step)
    │
    ├── GitHub Raw CDN  (data.json)
    │       └── session log, total hours completed
    │
    └── Cloudflare Worker  (edge-deployed, globally distributed)
            │
            ├── GET /                           → YouTube Data API v3
            │     Like counts across 18 tracked    (parallel fetch, KV-cached 5 min)
            │     comments → hours owed
            │
            ├── /{ADMIN_SECRET}/focus-on|off    → KV write   [path-shortcut, legacy bookmark support]
            ├── /{ADMIN_SECRET}/sync            → delegates to syncTodoistCompleted
            │
            ├── ?action=adminLogin              → issues a signed 30-day session token
            ├── ?action=getFocusState           → KV read
            ├── ?action=setFocusState           → KV write  [admin]
            │
            ├── ?action=logSession              → Todoist task create+complete
            │     Instant manual logging           + immediate data.json commit  [admin]
            │     from the admin panel
            │
            ├── ?action=getTodoistTasks         → Todoist REST API v1
            │     Active tasks in the configured   (cursor-paginated, KV-cached 3 min —
            │     project, sorted by tier+priority   not currently rendered in the frontend)
            │
            ├── ?action=syncTodoistCompleted    → Todoist REST API v1
            │     Detects newly completed tasks,    → GitHub Contents API
            │     parses duration from description, → commits to data.json
            │     deduplicates via KV set
            │
            ├── ?action=getHoursHistory         → KV read (feeds the graph)
            ├── ?action=getRemainingHistory     → KV read (feeds the graph)
            ├── ?action=getConfig               → KV read
            └── ?action=setConfig               → KV write  [admin]
```

The browser holds no secrets. All credentials live exclusively in the Worker's environment (set via `wrangler secret put`, never committed).

## Features

### Hours Owed — YouTube Like Tracking

The Worker fetches like counts from 18 tracked YouTube comments using the YouTube Data API v3. Each comment has a `hoursPerLike` value set at the time it was added (matching the video length). The Worker multiplies likes × hoursPerLike across all comments and sums them into a total hours-owed figure.

To prevent inflated counts from stale YouTube API responses, stored like counts are **monotonically non-decreasing** — the Worker always takes `Math.max(apiCount, storedCount)`. If the API temporarily returns a lower number, the stored value holds.

Results are cached in Cloudflare KV for 5 minutes to stay well within the YouTube API's daily quota. The browser shows a live countdown to the next refresh.

### Hours Completed — Todoist Auto-Sync

Every 5 minutes (or on demand — see below), the Worker's `syncTodoistCompleted` action:

1. Fetches recently completed tasks across the whole Todoist account since the last sync
2. Parses a study duration from a `*` marker in the task description, falling back to scanning the task's comments — see **Duration marker formats** below
3. Determines the correct date (see Date Priority)
4. Appends the session(s) to `data.json` via a batched GitHub Contents API commit
5. Marks each task ID as processed in KV (`todoist-processed`) to prevent double-counting on future syncs

A distributed lock (`todoist-sync-lock` in KV, 60-second TTL) prevents race conditions if the sync is triggered concurrently.

#### Duration marker formats

The `*` marker must appear in the task description or a comment, followed by an hours/minutes/seconds combination — any subset of the three, in that order, any common spelling, spacing optional:

- Hours: `2h`, `2hr`, `2hrs`, `2 hour`, `2 hours`
- Minutes: `90m`, `90min`, `90mins`, `90 minute(s)`
- Seconds: `30s`, `30sec`, `30 seconds`
- Combined, with or without spaces: `2h30m`, `1h20m15s`, `45m30s`, `2 hours 15 min`

Decimals are allowed only on whichever unit ends up being the **last one present** — `2.5h` and `1h30.5m` both parse fine, but `2.5h30m` is rejected as ambiguous (is the `.5` hours or minutes?). A bare number with no unit, a zero-length duration, or no `*` marker at all all resolve to "no duration found" and the task is skipped by the sync.

### Date Priority

When deciding which date to log a session under:

1. **Task due date** — when the work was *scheduled* to happen (preferred)
2. **Completion timestamp** — when the task was actually marked done (fallback)

So if I complete Monday's assignment on Wednesday, it logs under Monday. The heatmap reflects when study *should have* happened, not just when I got around to finishing it. (Manually logged sessions from the admin panel always use today's IST date.)

### The Ledger — Admin Panel

A hidden panel, invisible to every regular visitor, lets me manage the tracker directly from the live site instead of only via a bookmarked URL:

- **Ctrl+Shift+A** opens/closes the panel; **Escape** closes it; **Ctrl+Shift+F** force-toggles focus/break directly, without opening the panel, as long as a valid session is cached.
- **Login**: one secret, no separate username. On success the Worker issues a signed, 30-day-expiring session token (HMAC-SHA256 via the Workers-native Web Crypto API, keyed off `ADMIN_SECRET` — no extra secret needed). The token is cached in `localStorage` so I don't retype the secret on every visit; it's sent as `?token=` (or in the request body for `logSession`) instead of the raw secret from then on.
- **Focus/break toggle**: a button reflecting live state — clicking it (or the keybind) flips focus on/off instantly.
- **Log completed session**: a small form (topic, hours, an optional `deep`/`light` label) that, on submit:
  1. Resolves (or creates, once) a dedicated Todoist project named **"The Ledger"**
  2. Creates a task there with the topic and a `* Xh` duration marker, and immediately marks it complete
  3. Commits the session straight to `data.json` in the same request — no waiting on the 5-minute poller
  4. Marks the new task's ID as already-processed in KV, so the periodic sync never double-counts it
  5. Updates the page's stats instantly (hero numbers, heatmap, recent-sessions list, graph) without waiting on GitHub's raw-CDN cache
- **Force Todoist sync**: triggers `syncTodoistCompleted` on demand instead of waiting for its 5-minute interval.
- **Log out** clears the cached session.

For backward compatibility, the original phone-bookmarkable URLs still work unchanged: `https://study-tracker-likes.mangalarush.workers.dev/{ADMIN_SECRET}/focus-on`, `.../focus-off`, and `.../sync` — these accept the raw secret directly, no login step required.

### Adaptive Sleep Notice

Between 12 AM and 8 AM IST, the clock card shows a contextual message:
- If a focus session is **active** → *"All-nighter mode — go get it!"* (green)
- If no focus session is running → *"Prolly sleeping — no updates 12 AM to 8 AM!"* (red)

Once the sleeping state is decided for a given night, it stays committed — it won't flicker back if I'm between focus sessions during a break. But if I start a new focus session, it overrides immediately.

### 90-Day Study Heatmap

A GitHub-style contribution grid covering the last 90 days. Each cell represents one day; colour intensity maps to hours studied that day across 6 levels (0 → 0–2h → 2–5h → 5–9h → 9–12h → 12h+). The grid is padded to full ISO weeks, only renders from the deploy date onwards, and each cell is hoverable with an exact hours tooltip. All date arithmetic runs in IST.

### Study Insights

- **Today's hours** — sessions logged for the current IST study day, plus day-of-year counter
- **This week's hours** — summed across the current ISO 8601 week, plus week number

"Today" is defined with a 2 AM cutoff — sessions logged before 2 AM are attributed to the previous calendar day, reflecting that late-night study belongs to the night it started.

### Hours Remaining Over Time

A hand-drawn canvas line graph plotting owed (red, dashed), done (green, dashed), and remaining (bright green, solid, gradient-filled) hours over time, with a shaded band between owed and done. Range toggles (today / 7d / 30d / 90d / all) and per-line visibility toggles (at least one line must stay visible) let you zoom in on recent movement or the whole history. Hovering snaps a tooltip to the nearest point on each visible line. Falls back to a "still building up history" placeholder until enough data points (`getHoursHistory` / `getRemainingHistory`) exist.

### Like Sources Breakdown

A collapsible panel showing each tracked YouTube comment that has earned at least 10 hours, with its like count, total hours earned, and a `+Xh` badge that links directly to the comment. Sorted by likes then hours-per-like. Clicking the badge or title opens the comment for one-click liking.

## Cloudflare KV Namespace

| Key | Stored value |
|---|---|
| `cacheV2` | Full worker response payload (likes, hours, breakdown) with timestamp |
| `counts` | Per-comment like counts — monotonically increasing |
| `lastChange` | Most recent like event: label, URL, from/to counts, IST timestamp |
| `hours-history` | Time series of total hours-owed snapshots (feeds the graph) |
| `remaining-history` | Time series of `{owed, done, remaining}` snapshots (feeds the graph) |
| `focus-state` | `{ active: bool, since: ISO, lastSeen: ISO }` |
| `todoist-cache` | Cached active task list with timestamp (`getTodoistTasks`) |
| `todoist-last-sync` | ISO timestamp of last completed-task sync window start |
| `todoist-processed` | Array of synced Todoist task IDs (deduplication set, capped at 1000) |
| `todoist-sync-lock` | Write lock — present means sync in progress; 60s TTL |
| `ledger-project-id` | Cached Todoist project id for "The Ledger" (admin-logged sessions) |
| `config-target-project` | Todoist project name(s) to filter `getTodoistTasks` against |

## Environment Variables (Worker)

| Variable | Used for |
|---|---|
| `YOUTUBE_API_KEY` | YouTube Data API v3 — comment like counts |
| `TODOIST_TOKEN` | Todoist REST API — active tasks, completions, and admin-panel task creation |
| `GITHUB_TOKEN` | GitHub PAT (`contents:write`) — committing `data.json` |
| `ADMIN_SECRET` | The one admin credential — checked directly for legacy bookmark URLs and `adminLogin`, and used as the HMAC key to sign/verify admin-panel session tokens |
| `KV` | Cloudflare KV namespace binding |

None of these are declared in `wrangler.toml` (only the KV binding is) — set them with `wrangler secret put <NAME>`, which uploads directly to the Worker without ever touching this repo.

## Refresh Cadence

| Data | Interval | Method |
|---|---|---|
| Like counts / hours owed | 5 min | Worker KV cache TTL |
| Active Todoist tasks (`getTodoistTasks`) | 3 min | Worker KV cache TTL |
| Todoist completed sync | 5 min, or instantly via the admin panel's "Force Todoist sync" button | `setInterval` / manual |
| Manually logged sessions (admin panel) | Instant | Direct `data.json` commit in the same request |
| Focus state | 60 sec (30 sec while active) | `setInterval` |
| Clock display | 1 sec | `setInterval` |

## `data.json` Schema

```json
{
  "hoursCompleted": 341.45,
  "sessions": [
    {
      "date": "27 Jun 2026",
      "topic": "Watch: Decimal to Binary, Octal and HexaDecimal Conversion",
      "hours": 0.15,
      "labels": ["deep"]
    }
  ]
}
```

`hoursCompleted` is recomputed as the sum of all session hours on every write. `date` is the human-readable IST date used for display (`D MMM YYYY`). `labels` is a free-form string array — currently used for `deep`/`light` distinctions, defaults to `[]`. Older entries predate the `labels` field and simply omit it. The frontend also defensively checks for `loggedAt`/`dateISO` fields on each session for sorting, but no current entry actually has them — that's forward-compatible dead code, not a documented part of the schema yet.

## Local Development

No build step, no `npm install` for the frontend.

```bash
# Serve the static page
npx serve .
# or
python3 -m http.server 8080
```

### Running/deploying the Worker

`worker.js` and `wrangler.toml` are version-controlled in this repo (the KV namespace binding and account id are in `wrangler.toml`; the actual secrets are not — they live only on Cloudflare).

```bash
npm install -g wrangler
wrangler login                 # once, per machine

# Validate without deploying
wrangler deploy --dry-run

# Deploy for real
wrangler deploy

# Set/rotate a secret (prompts for the value, never touches this repo)
wrangler secret put ADMIN_SECRET
wrangler secret put TODOIST_TOKEN
wrangler secret put GITHUB_TOKEN
wrangler secret put YOUTUBE_API_KEY
```

Rotating `ADMIN_SECRET` invalidates every cached admin-panel session token (they're signed with it) and any legacy bookmarked URLs using the old value — just log in again with the new secret.

To point the frontend at a local Worker instead of the deployed one, run `wrangler dev` and update `WORKER_URL` in `index.html` to `http://localhost:8787`.

## Enabling Maintenance Mode

Uncomment this block near the top of `index.html`:

```html
<script>
  document.addEventListener("DOMContentLoaded", function () {
    document.getElementById("maintenance-screen").style.display = "flex";
    document.body.style.overflow = "hidden";
  });
</script>
```

This shows a full-screen animated overlay. Re-comment to restore the live page. No other changes needed.

## Stack

| Layer | Technology |
|---|---|
| Frontend | Vanilla HTML, CSS, JavaScript — zero dependencies, zero build step |
| Hosting | GitHub Pages |
| Edge compute | Cloudflare Workers (V8 isolates, globally distributed) |
| Deploy tooling | Wrangler CLI (`wrangler.toml` in-repo; secrets set out-of-band) |
| Persistent state | Cloudflare KV (key-value store at the edge) |
| Session data | GitHub Contents API — `data.json` committed directly |
| Task management | Todoist REST API v1 |
| Like tracking | YouTube Data API v3 |
| Admin auth | Signed HMAC-SHA256 session tokens (Workers Web Crypto API) + legacy shared-secret fallback |
| Timezone | IST (Asia/Kolkata) throughout, including all date arithmetic |

The live focus dot means anyone can visit and see whether studying is actually happening right now.

Every part of this started as a real problem I had. The code exists because I needed it to.
